# 产品评价系统指标计算说明

本文档基于当前前端页面中实际使用的指标整理，说明 `尊享B5.xlsx`、`尊享C3.xlsx` 两份原始数据如何计算产品评价系统所需指标。

## 1. Excel 原始数据结构

### 尊享B5.xlsx

主要原始数据页为 `Sheet1`：

| 字段 | 用途 |
| --- | --- |
| 日期 | 时序日期，按升序排列 |
| 净值 | 产品累计净值 |
| 基准净值 | 产品基准累计净值 |
| 现金 | 现金策略权重 |
| 债券策略、商品套利、套利、指数增强、市场中性、CTA、主观选股、货币市场型、股票T0、量化多头、宏观对冲、多策略 | 各策略权重 |

辅助净值页为 `Sheet2`：

| 字段 | 用途 |
| --- | --- |
| 发布日期 | 净值发布日期 |
| 产品净值 | 产品累计净值 |
| 60%shibor+40%沪深300 | 基准累计净值 |

### 尊享C3.xlsx

主要原始数据页为 `Sheet2`：

| 字段 | 用途 |
| --- | --- |
| 日期 | 时序日期，按升序排列 |
| 净值 | 产品累计净值 |
| 基准净值 | 产品基准累计净值 |
| 现金 | 现金策略权重 |
| CTA、指数增强、市场中性、货币市场型、可转债、量化多头、主观选股、套利、债券策略、股票多头、宏观对冲、多策略 | 各策略权重 |

两份 Excel 已预置 `时序计算`、`指标计算`、`指标说明` 工作表。若重新在系统中计算，建议以原始数据页为唯一输入，计算逻辑与这些工作表保持一致。

## 2. 通用中间序列

以下序列是后续 UI 指标的基础。

| 中间字段 | 计算方式 |
| --- | --- |
| 产品区间收益 `r_t` | `本期净值 / 上期净值 - 1`，首期为空 |
| 基准区间收益 `rb_t` | `本期基准净值 / 上期基准净值 - 1`，首期为空 |
| 区间超额收益 `excess_t` | `产品区间收益 - 基准区间收益` |
| 产品累计最高净值 `peak_t` | `MAX(起始日至 t 日产品净值)` |
| 产品回撤 `dd_t` | `产品净值_t / peak_t - 1` |
| 基准累计最高净值 | `MAX(起始日至 t 日基准净值)` |
| 基准回撤 | `基准净值_t / 基准累计最高净值_t - 1` |
| 单期最大调仓 | `MAX(ABS(本期各策略权重 - 上期各策略权重))` |
| 单期换手 | `SUM(ABS(本期各策略权重 - 上期各策略权重)) / 2` |
| 策略集中度 HHI | `SUMSQ(当期各策略权重)` |
| 现金仓位 | `现金` 列 |
| 年化频率 | `365 / 平均相邻日期间隔` |

说明：两份 Excel 的净值频率接近周频，但年化口径使用实际日期间隔，因此不要固定写死为 52。

## 3. 产品基础信息

| 前端字段 | UI 展示位置 | 计算方式 / 来源 |
| --- | --- | --- |
| 产品名称 | 产品池、详情页、对比页 | 来自文件名或 `指标计算` 页的“产品名称”，如 `尊享B5`、`尊享C3` |
| 产品代码 | 产品池、详情页、对比页 | 两份 Excel 未提供，需要额外维护产品主数据 |
| 管理人 | 产品池、详情页、管理人分析 | 两份 Excel 未提供，需要额外维护产品主数据 |
| 产品分类 | 产品池筛选、预警大类 | 两份 Excel 未提供，需要额外维护分类映射，如积极看多、谨慎看多、低相关、类固收 |
| 最新日期 | 详情页、净值图 | `最后一条日期` |
| 最新净值 | 详情页、抽屉 | `最后一期产品净值` |
| 最新基准净值 | 详情页净值图 | `最后一期基准净值` |

## 4. 产品收益指标

| 指标 | UI 展示位置 | 计算方式 |
| --- | --- | --- |
| 累计净值 | 产品详情“收益指标” | `最新净值` |
| 本周净值涨跌 | 产品详情“收益指标” | `最新净值 / 上一期净值 - 1`；若要严格周口径，可取最新日期前 5 个自然日或最近一个周频观测点 |
| 成立以来净值涨跌 / 累计收益 | 产品详情、抽屉 | `最新净值 / 起始净值 - 1` |
| 基准累计收益 | 净值图、超额分析 | `最新基准净值 / 起始基准净值 - 1` |
| 累计超额收益 | 详情抽屉、后续归因 | `累计收益 - 基准累计收益` |
| 年化收益 | 产品池、详情页、对比页 | `(1 + 累计收益) ^ (365 / (最新日期 - 起始日期)) - 1` |
| 基准年化收益 | 管理人/产品基准对照 | `(1 + 基准累计收益) ^ (365 / (最新日期 - 起始日期)) - 1` |
| 年化超额收益 | 信息比率、超额评价 | `年化收益 - 基准年化收益` |
| 相对基准胜率 | 可用于管理人/产品胜率 | `COUNTIF(区间超额收益 > 0) / COUNT(区间超额收益)` |

## 5. 产品风险指标

| 指标 | UI 展示位置 | 计算方式 |
| --- | --- | --- |
| 年化波动率 | 产品池、详情页、对比页 | `STDEV.S(产品区间收益) * SQRT(年化频率)` |
| 基准年化波动率 | 基准评价 | `STDEV.S(基准区间收益) * SQRT(年化频率)` |
| 最大回撤 | 产品池、详情页、对比页、预警 | `MIN(产品回撤序列)` |
| 当前回撤 | 产品详情、抽屉 | `最新净值 / 历史最高净值 - 1` |
| 回撤观察期 / 恢复天数 | 抽屉、预警 | `最新日期 - 最近一次产品净值创新高日期` |
| 夏普比率 | 产品池、详情页、对比页 | `(年化收益 - 无风险利率) / 年化波动率`；当前 Excel 使用无风险利率 `2.50%` |
| 卡玛比率 | 产品详情、对比页、预警 | `年化收益 / ABS(最大回撤)` |
| 跟踪误差 | 后续超额质量指标 | `STDEV.S(区间超额收益) * SQRT(年化频率)` |
| 信息比率 | 后续超额质量指标 | `年化超额收益 / 跟踪误差` |

## 6. 策略行为指标

| 指标 | UI 展示位置 | 计算方式 |
| --- | --- | --- |
| 配置变化曲线 | 产品详情“策略行为” | 逐期读取各策略权重列，按日期形成堆叠面积图 |
| 单期最大调仓 | 产品详情“策略指标” | `MAX(单期最大调仓序列)` |
| 累计调仓幅度 | 产品详情“策略指标” | `SUM(单期换手序列)` |
| 策略集中度 HHI | 产品详情“策略指标” | `最新一期 SUMSQ(各策略权重)` |
| 平均现金仓位 / 现金缓冲度 | 产品详情“策略指标” | `AVERAGE(现金仓位序列)` |
| 最新现金仓位 | 可用于风险解释 | `最新一期现金权重` |

## 7. 产品池与对比页汇总指标

这些指标不是单个 Excel 内部指标，而是产品集合层面的聚合。

| 指标 | 计算方式 |
| --- | --- |
| 产品总数 | 产品主数据中产品数量 |
| 管理人数量 | 产品主数据中去重管理人数量 |
| 分类产品数 | `COUNTIF(产品分类 = 当前分类)` |
| 特色产品数 | `COUNTIF(特色标签 = 当前特色系列)`，需额外维护特色标签 |
| 平均年化收益 | 当前分类下 `AVERAGE(各产品年化收益)` |
| 平均最大回撤 | 当前分类下 `AVERAGE(各产品最大回撤)` |
| 平均年化波动率 | 当前分类下 `AVERAGE(各产品年化波动率)` |
| 平均夏普比率 | 当前分类下 `AVERAGE(各产品夏普比率)` |
| 产品对比表 | 对被选产品展示年化收益、最大回撤、年化波动、夏普、卡玛 |

现有两份 Excel 只能形成 2 只产品的集合指标。若要完整支撑前端产品池，需要补充所有产品的同结构 Excel 或统一后的产品净值/策略权重表。

## 8. 基准、同类与管理人指标

| 指标 | 当前前端使用方式 | 两份 Excel 可计算程度 |
| --- | --- | --- |
| 产品基准净值曲线 | 产品详情“产品 vs 基准” | 可直接使用 `基准净值` 列 |
| 产品相对基准超额 | 点击净值点展示 | `产品净值_t / 基准净值_t - 1`，或使用累计/年化超额收益 |
| 同类中位数净值 | 产品详情“产品 vs 同类中位数” | 两份 Excel 不足以计算，需要同类产品池净值；口径为同分类产品每期收益或净值的中位数合成 |
| 同类回撤 | 抽屉“同类回撤” | 需要同类产品池；可取同类产品最大回撤的中位数或平均值 |
| 管理人基准净值 | 管理人“战略配置能力” | 单管理人若有年度中枢权重与资产指数收益，可按 `SUM(年度中枢权重 × 对应资产指数收益)` 合成；两份 Excel 只有产品策略权重，不能完整代表管理人战略中枢 |
| 管理人基准最大回撤 | 管理人总览 | 对管理人基准净值计算最大回撤 |
| 管理人滚动夏普 60D | 管理人战略页 | 对管理人基准区间收益滚动 60 个观测点计算夏普 |
| 环境匹配度 | 管理人战术页 | 需要市场环境标签和战术权重标签；两份 Excel 不足 |
| 环境胜率 | 管理人战术页 | 需要按环境分组后的产品/组合超额收益 |
| 环境超额收益 | 管理人战术页 | 需要环境窗口内产品收益与基准收益 |
| 事件响应度 | 管理人战术页 | 需要事件发生日、调仓/风控动作日期、响应 SLA |
| 事件胜率 | 管理人战术页 | 需要事件后观察窗口收益或超额收益 |
| 事件超额收益 | 管理人战术页 | 事件后产品收益 - 基准收益 |
| 覆盖底基、管理人集中度 | 管理人选基能力 | 需要底层基金持仓或产品主数据，两份 Excel 不足 |

## 9. 预警与 SOP 指标

当前前端包含风险事件预警和售后 SOP。两份 Excel 可以直接产生“产品驱动”的部分预警，投研驱动和客户触达状态需要额外事件表。

### 可由 Excel 直接触发的产品驱动预警

| 预警场景 | 计算方式 |
| --- | --- |
| 最大回撤超阈值 | `最大回撤 <= -阈值`，系统设置示例阈值为 `-6%` |
| 净值单日/单期异常下跌 | `产品区间收益 <= -阈值`，系统设置示例为 `-1.5%` |
| 回撤恢复期延长 | `当前回撤 < 0` 且 `回撤观察期 > 阈值天数` |
| 卡玛明显恶化 | 连续观察窗口内 `卡玛比率 < 阈值`，系统示例为连续 2 个月低于 0.6 |
| 波动率上升 | `近 N 期年化波动率 / 长期年化波动率 >= 1.6` |
| 规模快速下降 | 两份 Excel 不含规模，需要补充规模/AUM 序列；口径可为 `近 30 日规模变化 <= -15%` |

### 投研驱动预警

前端 `data.js` 中配置了大类阈值规则，例如：

| 大类 | 示例阈值 | 需要补充的数据 |
| --- | --- | --- |
| 积极看多 | 量化指增单周回撤 >= 5%；最大回撤 >= 20%；宽基指数单日跌幅 >= 3% | 产品策略类型、宽基指数行情 |
| 谨慎看多 | 多策略单周回撤：稳健 >= 2%、均衡 >= 3%；10Y 国债期货单日价格持续下跌 >= 1% | 产品策略标签、债券/股指行情 |
| 低相关 | CTA 单周回撤 >= 3%；套利单周回撤 >= 2.5%；商品板块单日跌幅 >= 5% | CTA/套利产品收益、商品行情 |
| 类固收 | 单周回撤 >= 1%；10Y 国债期货单日下跌 >= 1% | 类固收产品净值、债券行情 |

### SOP 任务指标

| 指标 | 计算方式 | 数据要求 |
| --- | --- | --- |
| 高/中/低等级预警数 | `COUNTIF(事件等级)` | 事件表 |
| 预警总数 | `COUNT(事件)` | 事件表 |
| 投研驱动数量 | `COUNTIF(驱动 = 投研驱动)` | 事件表 |
| 待一线确认 | `COUNTIF(stage = 0)` | SOP 状态表 |
| 待客户触达 | `COUNTIF(stage = 1)` | SOP 状态表 |
| 可营销机会 | `COUNTIF(tag = 增值)` | 事件表 |
| 今日到期任务 | `COUNTIF(到期日 = 今日 且 未完成)` | SOP 状态表，需要到期日 |
| 紧急处置清单 | `等级=高` 或 `等级=中 且 stage < 2` 后排序 | 事件表 + SOP 状态表 |

## 10. 建议的数据落表结构

为了让前端从模拟数据切换到真实数据，建议把两份 Excel 先统一成以下结构。

### 产品主表

`product_id, product_name, product_code, manager_id, manager_name, category, rating, featured_tag`

### 产品时序表

`product_id, date, nav, benchmark_nav`

### 策略权重表

`product_id, date, strategy_name, weight`

### 产品指标表

`product_id, start_date, latest_date, latest_nav, latest_benchmark_nav, cumulative_return, annual_return, benchmark_annual_return, annual_excess_return, annual_volatility, max_drawdown, current_drawdown, recovery_days, sharpe, calmar, tracking_error, information_ratio, benchmark_win_rate, max_single_adjustment, cumulative_turnover, hhi, average_cash_weight, latest_cash_weight`

### 事件与 SOP 表

`event_id, product_id, category, scene, driver, tag, level, reason, action, stage, due_date, created_at, updated_at`
