# 网页逆向生成 Prompt

下面这段 prompt 用于从零生成当前网页。你可以在此基础上继续修改业务口径、页面模块、数据字段或交互要求。

```text
请生成一个中文的私募/信托产品评价系统静态前端网页，名称为“产品评价系统”。整体形式是一个无需后端、无需构建工具即可运行的单页应用，使用 HTML、CSS、原生 JavaScript 实现。页面入口为 `index.html`，样式为 `styles.css`，业务逻辑可拆分为 `app.js` 和 `src/` 下的若干 JS 文件。整体视觉风格要求专业、克制、偏金融投研系统。

## 总体布局

页面采用左侧固定导航栏 + 右侧主内容区布局。

左侧导航栏标题为“产品评价系统”，包含以下一级菜单和二级菜单：

1. 产品池
   - 产品详情
   - 产品对比
2. 管理人分析
   - 策略配置能力
   - 战术配置能力
   - 选基能力
3. 预警和售后
   - 风险事件预警
   - 售后SOP
4. 系统设置

右侧顶部栏包含：
- 当前页面标题
- 面包屑：首页 / 当前页面
- 全局搜索框，placeholder 为“搜索产品名称、管理人、代码”
- 数据截至：2026-05-20
- 更新于：2026-05-21 09:30
- 刷新按钮、通知按钮、用户按钮“张三”

所有页面在同一个单页应用中切换，不刷新浏览器页面。点击信息卡、指标卡、图表或状态时，右侧弹出抽屉展示详情说明。操作完成时使用 toast 提示。

## 产品池页面

产品池页面顶部为“产品池概览”，内容要简练

概览下方必须分成三个模块：

### 1. 产品分类

展示四个分类卡片：

- 积极看多
- 谨慎看多
- 低相关
- 类固收

每张卡片展示：
- 分类名称
- 该分类下产品数量，例如“5只”
- 数字和箭头形式，展示叫上个周期增加或减少几只

不要在分类卡片中展示任何平均评分。


点击分类后，产品列表和产品指标区域同步切换到该分类下的产品。

### 2. 特色产品

展示三个特色产品卡片：

- 磐石3.0
- 类全委
- 白金类全委

每张卡片展示：
- 分类名称
- 该分类下产品数量，例如“5只”
- 数字和箭头形式，展示叫上个周期增加或减少几只


### 3. 产品指标

展示当前选中产品分类下产品的平均指标，必须包含：

- 平均年化收益
- 平均最大回撤
- 平均波动率
- 平均夏普比率

指标区域必须和产品分类联动。选择哪个分类，就只用该分类下产品计算平均值。每个指标卡显示指标名称、均值。

产品池下方为产品列表，支持：
- 按分类筛选
- 按管理人筛选
- 选择产品
- 导入产品对比

表格列包含：
- 选择
- 产品
- 分类
- 管理人
- 年化收益
- 最大回撤
- 夏普
- 操作

## 管理人分析页面

管理人分析顶部展示管理人信息：
- 管理人名称
- 管理人类型
- 产品数量


### 策略配置能力

策略配置能力部分必须包含三张图和一个定性分析模块。

#### 图 1：基准净值曲线

根据各管理人年度战略配置中枢权重，分别乘以对应资产类别指数收益，合成每个管理人的战略配置基准组合收益，并累计生成净值曲线。

要求：
- 横轴为日期
- 纵轴为净值
- 不同管理人用不同颜色线条展示

#### 图 2：基准回撤曲线

基于上述战略配置基准净值曲线，计算每个管理人的历史回撤，即当前净值相对历史最高净值的跌幅。

要求：
- 横轴为日期
- 纵轴为回撤比例
- 不同管理人用不同颜色线条展示

#### 图 3：滚动夏普曲线

基于每个管理人的战略配置基准组合收益，计算滚动窗口夏普比率，例如 12 周或 6 个月滚动夏普。

要求：
- 横轴为日期
- 纵轴为滚动夏普比率
- 不同管理人用不同颜色线条展示

#### 定性分析

在三张图下方展示“定性分析”模块，说明：
战略配置基准用于回答“年度中枢本身是否有效”。净值曲线看长期收益贡献，回撤曲线看中枢在压力阶段的防守能力，滚动夏普看风险收益效率是否稳定。

### 战术配置能力

战术配置能力部分完全改为“六个指标 + 定性分析模块”。

六个指标分成两个模块：

#### 环境模块

指标：
- 环境匹配度
- 胜率
- 超额收益

#### 事件模块

指标：
- 事件响应度
- 胜率
- 超额收益

六个指标用卡片展示。分为两个大模块，每张卡片包含：

- 指标名
- 指标值


下方展示“定性分析”模块，说明：
战术配置评价重点从单纯调仓频率转向“是否看对环境、是否及时响应事件、响应后是否产生胜率和超额收益”。环境模块适合观察市场状态判断，事件模块适合观察回撤、风格漂移、开放赎回或业绩突变后的处理质量。

### 选基能力

展示空态：
“选基能力待接入底层基金数据后展示”。




## 预警和售后

预警和售后包含两个主区域：

### 风险事件预警


指标卡包含：
- 最大回撤超阈值
- 净值异常下跌
- 波动率突然上升
- 夏普持续下降
- 卡玛明显恶化
- 规模快速下降

### 售后SOP


指标卡包含：
- 待一线确认
- 待客户触达
- 可营销机会
- 今日到期任务

下方展示强关注列表，列包含：
- 产品
- 场景
- 驱动：产品驱动或投研驱动
- 标签：预警，安抚，增值等
- 等级
- 触发原因
- 建议动作
- 状态

每个事件状态旁边有“推进”按钮，点击后状态向下一阶段流转，并显示 toast。

## 图表要求

使用原生 SVG 绘制图表，不依赖第三方图表库。

需要实现：
- 折线图
- 柱状图
- 面积堆叠图
- 横向百分比图或紧凑指标图

折线图需要支持：
- 多序列
- 百分比格式
- 自定义最小最大值
- 阈值线
- 横轴日期标签
